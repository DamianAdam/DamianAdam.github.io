Hexo 博客使用技巧

本文最后更新于：2 小时前

Hexo + GitHub 搭建个人博客

01. 工具准备

  • 推荐在Windows环境下执行，Linux下总出错。
  • 确保安装好NodeJS、npm和Git。（node版本要大于10）
  • 安装Hexo。
  • 通过检查版本号的方式验证安装。

02. 创建 GitHub 共有仓库

  • 在Github中New一个仓库，有这样一些要求：
    • Repository name：Github用户名.github.io
    • 仓库属性：Public
    • Add a README file

  • 完成后点击Create repository创建即可。

03. 配置SSH密钥

  • 在任意地方创建一个Blog目录，然后进入这个Blog目录，右键打开Git Bash。（之后有用）
  • 然后在C:\Users\Administrator\.ssh目录下就会出现一个id_rsa.pub文件，使用记事本或其他工具打开，复制其中的内容。
    • Administrator替换为自己Windows的用户。
  • 进入https://github.com/settings/keys，New SSH key。
    • Title：建议填写HexoBlog，以便区分。
    • Key：id_rsa.pub中的内容。

  • 然后回到Git Bash中，使用命令验证安装，提示successfully即为成功。

04. 本地生成博客页面

  • 在Blog目录中打开Git Bash，初始化Hexo。
  • 初始化完成后，Blog目录下会自动生成一些文件。

  • 修改_config.yml中Site的内容。（不做也可以）
    • title：博客标题
    • subtitle：博客标语
    • description: 描述信息
    • author：作者
    • language：语言
    • timezone：时区
  • 静态生成本地页面。
  • 此时，浏览器中访问http://localhost:4000/即可看到页面了。
    • 看到页面后，即可返回Git Bush按Ctrl + C关闭本地服务器。

05. 发布博客到互联网（上传到Github）

  • 修改_config.yml中deploy的内容。（必须要做）
    • type：固定为git。
    • repo：仓库的HTTPS地址，在仓库的Code中可以看见。（每个人不一样）
    • branch: 分支，写main即可。（有些教程这里为master，那是Github老版本的主分支，现在还写master会出错）
  • 返回Blog的Git Bash，安装自动化部署发布工具。
  • 生成页面并发布页面。（碰见OpenSSL基本上是网络问题，多试几次即可）
  • 当Github中的仓库出现文件时，说明博客已经发布成功。
  • 此时访问https://github用户名.github.io/即可访问到你的博客。

更换 Hexo 个人博客的主题

  • 访问 Hexo 官网：Hexo官网，点击 Themes 。在这里可以选择所有可用的主题，我们在 Search 中搜索 fluid ，然后点击标题，它会跳转到 Github 项目界面，根据里面的说明下载主题与依赖。
  • 修改 Hexo 根目录下的 _config.yml 文件，将 theme 的值改为 fluid 。
  • 根据主题官网给出的文档自行配置网站需要开启的功能。
  • 清理、生成、发布。

使用 Hexo 发表文章

  • 首先，在 Blog 目录下打开 Git Bash，然后使用 hexo new post 命令创建一篇 Markdown 文章。
  • 接着，到 Blog\source\_posts 目录下编辑对应的 .md 文件。
  • 如果这篇文章有标签，请在开头处添加配置信息。
  • 本地部署。
  • 发布于上传。
  • 文章顶部配置内容参考手册：Front-matter
